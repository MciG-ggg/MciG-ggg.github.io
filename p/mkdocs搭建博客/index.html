  1. 让我们本地预览一下：
1

mkdocs serve | cat

现在你可以通过访问 http://127.0.0.1:8000 来预览你的博客。

要部署到 GitHub Pages，你需要：

  1. 首先提交所有更改：
1

git add . && git commit -m "使用 MkDocs 重构博客" | cat
  1. 使用 MkDocs 部署到 GitHub Pages：
1

mkdocs gh-deploy | cat

现在你的博客已经搭建完成！以下是一些重要信息：

  1. 博客结构：

    • docs/ - 所有的文档文件
    • docs/index.md - 首页
    • docs/about.md - 关于页面
    • docs/tech/ - 技术博客目录

  2. 如何写新博客：

    • 在 docs/tech/ 目录下创建新的 .md 文件
    • 在 mkdocs.yml 的 nav 部分添加新文章的链接

  3. 本地预览：

    1

    mkdocs serve

  4. 部署更新：

    1

    mkdocs gh-deploy

  5. 特色功能：

    • 响应式设计
    • 深色/浅色模式切换
    • 搜索功能
    • 代码高亮
    • 目录导航

要写新文章，你可以：

  1. 在 docs/tech/ 目录下创建新的 .md 文件
  2. 在 mkdocs.yml 的 nav 部分添加文章链接
  3. 使用 Markdown 格式编写文章
  4. 本地预览确认无误后部署

需要注意的警告：

  1. 有一些文件路径的警告，这是因为我们还没有创建 first-blog.md 文件
  2. 导航配置中的 技术/tech/index.md 路径需要修正
